# 文体：议论性（Argumentative）

## 目的
提出一个主张（论点），用推理与证据说服读者接受，而非仅仅告知或描述。

## 结构
1. **提出论点**：开篇亮明核心观点，避免绕弯。
2. **展开论证**：分论点 + 论据（事实 / 数据 / 案例 / 逻辑推理）。
3. **反驳异见**：预判并回应反方最有力的 1–2 个论点。
4. **结论**：回扣论点，给出判断或行动建议。

## 写法要点
- 一个段落服务一个分论点，段首可放「观点句」。
- 论据必须可查：数据标注来源，案例具体；无来源的观点要明确标为「我认为」。
- 推理链完整：前提 → 推理 → 结论，避免跳跃。
- 承认边界与不确定性，反而增强可信度。

## 宜
- 有清晰立场，不骑墙。
- 用「因为 / 所以 / 然而 / 可见」等逻辑连接。
- 结论有力但克制，不喊口号。

## 忌
- 只有态度没有论证（「XX 就是好」）。
- 以情绪代替逻辑、以个例代替普遍。
- 编造或夸大数据；偷换概念、稻草人谬误。
