# 文体：说明型（Expository）

## 目的
向读者清晰、准确地解释「是什么 / 怎么做 / 为什么这样」，传知而非说服或抒情。

## 结构
1. **引入**：点明要说明的对象与范围。
2. **分点阐明**：按定义 → 分类 → 机制/步骤 → 举例 → 比较的顺序展开。
3. **小结**：回顾要点，必要时给出使用或注意建议。

## 写法要点
- 以读者「已知」为起点，逐步引入「未知」；新术语首次出现即给出解释。
- 多用**定义、分类、举例、比较、类比、流程图**降低理解门槛。
- 逻辑顺序清晰：时间顺序、空间顺序、由总到分、由因到果皆可。
- 信息密度高但不堆砌；一段只讲一件事。

## 宜
- 客观、准确、条理分明。
- 用具体数字与可验证事实。
- 适当用列表、小标题、代码块、表格。

## 忌
- 夹带主观评价、情绪渲染、立场争论。
- 概念未定义就使用。
- 比喻喧宾夺主，淹没了说明对象本身。
