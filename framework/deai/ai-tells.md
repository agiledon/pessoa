# AI 痕迹目录（ai-tells）

> 本文件供 `deai.md` 的 **Pass 4 词汇去痕** 调用。它只列"机器腔"痕迹，**与文体、风格正交**：
> 正式论文里出现这些痕迹，要改；但改法是"正式但具体/有节奏"，不是把论文写成口语。
> 每条尽量给 `改前 → 改后`，便于照着替换。

---

## 一、中文高频禁用词

### 1.1 词汇腔（空转、堆概念）
| 痕迹词 | 改后 |
| --- | --- |
| 底层逻辑 | 根本道理 / 底层原因 |
| 认知升级 | 想明白 / 换个想法 |
| 闭环 | 完整一圈 / 自洽的回路 |
| 关键抓手 | 关键着力点 / 下手处 |
| 长期主义 | 看得长远 / 愿意等 |
| 价值闭环 | 赚钱的逻辑 / 自运转的循环 |
| 抓手 / 组合拳 | 办法 / 一套做法 |

### 1.2 过渡与表态套话
| 痕迹 | 改后 / 处理 |
| --- | --- |
| 此外 / 另外 / 与此同时 | 删掉，或换"还有一点" |
| 然而 / 但是（段首高频） | 删掉或并入前句 |
| 总而言之 / 综上所述 / 总结来说 | 删掉，结尾直接收 |
| 值得注意的是 / 值得一提的是 | 删掉，或"有意思的是" |
| 毋庸置疑 / 不可否认 | 删掉，直接说结论 |
| 众所周知 | 删掉，别替读者认定 |
| 在当今…时代 / 在…的背景下 | 删掉开场铺垫，直接切入 |

### 1.3 意义膨胀（把普通事实强行戴帽子）
| 痕迹 | 改后 |
| --- | --- |
| 标志着…的到来 | 就是…开始了 |
| 奠定了…的基础 | 是…的起点 |
| 凸显了…的重要性 | 说明…确实重要 |
| 具有里程碑意义 | 这一步很关键 |
| 成为…的关键转折点 | 从这里开始变了 |

### 1.4 宣传腔 / 空洞赞美
删掉或换具体：充满活力的、丰富的、深刻的、开创性的、令人叹为观止的、无与伦比的、革命性的。
例：`提供了无缝、直观、强大的体验` → `用起来顺手，上手快`。

### 1.5 模糊归因（冒充有来源）
| 痕迹 | 处理 |
| --- | --- |
| 专家认为 | 说出哪个专家 / 哪篇，否则删 |
| 行业报告显示 | 给出报告名与年份，否则删 |
| 相关研究表明 / 有学者指出 | 给具体文献，否则删 |
| 数据显示（无出处） | 标 `[需补充真实数据]` |

## 二、结构套路（与文体无关，所有类型都查）

- **否定式排比**：「这不是 X，而是 Y」「不仅仅是 A，更是 B」——偶尔用一次可以，连.Back to back 用就像同一模具。改：直接说 Y，或换成平实对比。
- **三段式 / rule of three**：强行把想法分成三组（"创新、效率、增长"）。改：两组也行，四组也行，或揉进一句。
- **虚假范围**：「从 X 到 Y」但 X、Y 无意义（"从个人到企业"）。改：删，或给真实跨度。
- **公式化"挑战与展望"**：结尾必写"面临挑战，未来可期"。改：删，或写一条具体下一步。
- **开头套路**：「随着…的发展」「近年来…」。改：直接从具体问题/现象起笔。
- **结尾套路**：「希望这对你有帮助」「总之，掌握 X 很重要」。改：删，或用一句有立场的话收尾。
- **平行罗列连接词套路（罗列腔）**：用「同样需要 / 也应当 / 都离不开 / 同样……」把多个并列论点用**同一套连接词**串成排比，句间只有格式连接、没有具体联系，是 AI 罗列腔；常用「故而，」「这些……」做生硬过渡。改：换用「譬如说 / 可以认为 / 与其…不如…」等自然转场，并在句间建立具体因果或类比，而非套同一个连接模板。
- **起笔无主句 / 不回指**：开头用「继承的是…」「需要注意的是…」「毋庸置疑…」这类无主语、不接前文的硬转折，读者不知在接什么。改：先交代回指对象（如「对 X 的继承，实则是对 Y 的继承」），再展开。

## 三、格式痕迹

- **破折号（——）滥用 +「这些」回指**：AI 最爱用破折号强行连接两个分句，且常用「——这些在…仍然成立 / 有效」引一长串列举再下套话结论。克制：去掉破折号，用「包括…在…仍然有效」自然收束，或拆成两句。
- **粗体滥用**：每段都加粗"关键词"。克制：一屏最多 1–2 处。
- **聊天记录残留**：「希望这对您有帮助」「如果您有任何问题…」「作为一个 AI…」。删。
- **弯引号**：中文正文用 `「」`，不用英文 `" "`；代码/外文引用除外。
- **中英混排腔（AI 痕迹，纠正旧规）**：中文为主，术语先译中文、括号注全英文（如 模型服务化（Model Serving）、规格（Spec）、提示词）；专有名词（Kubernetes、GitOps、CI/CD）保留英文即可。**不要在中文字与英文词之间为「分隔」而强行加空格**（如「模型 Serving」「Agent 运行时」「Prompt、Spec」）——那是 AI 排版腔。数字与中文之间（「5 年」）可酌情，但不要机械地给每个夹在中文里的英文词两侧都留空格。
- **标题层级**：最多三级，别无限细分；项目符号别堆成"关键词：解释"的固定模板。

## 四、英文 slop 黑名单（模型可能混出）

delve, leverage, utilize, facilitate, tapestry, landscape, realm, journey, beacon,
cornerstone, crucial, pivotal, testament, robust, navigate, unlock, empower, seamless,
revolutionary, cutting-edge, furthermore, moreover, additionally, "in today's",
"at its core", "at the heart of", "it's important to note that", "when it comes to",
"ultimately", "in conclusion"。
处理：换成简单具体的词；"in today's X" 类开场直接删。

## 五、反向黑名单（"装人味"比 AI 味更尴尬，禁用）

去 AI 不是表演人味。以下做法**禁止**，因为它们制造另一种模板感：
- 故意加错别字、拼音梗、网络脏话来"显得真实"。
- 把所有句子强行改短、全改成口语。
- 把所有判断都改成"我不太确定"，真人有立场时照样明确。
- 硬塞 emoji、口头禅、段子。
- 把一种痕迹换成另一种（如删了"此外"却加上一堆"说实话""讲真"当过渡）。

> 原则：文本已经自然，就不动它；改的是机器腔，不是人的声音。

## 六、before / after 速查

- 改前：`这个新功能作为我们团队致力于创新的证明。此外，它展示了我们在不断演变的技术格局中的关键作用。`
  改后：`新功能加了批处理、快捷键和离线模式。内测用户反馈大多说活干得更快了。`
- 改前：`这不仅是一个工具，更是一场思维方式的革命。`
  改后：`这工具最实用的地方，是它逼着你换一种方式想问题。`
- 改前：`值得注意的是，随着大模型的发展，相关内容生产发生了深刻变化。`
  改后：`大模型出来后，内容生产确实变了样——最明显的是……`
- 改前：`专家认为，该方案具有里程碑意义。`
  改后：`（删）该方案最关键的一步是……` 或 `据 XX 2025 年的报告，该方案……`
- 改前：`继承的是"怎么可靠运维大规模分布式系统"的方法论。声明式期望状态、GitOps、多租户隔离、…——这些在 AI 时代仍然成立。Agent 运行时与模型 Serving 同样需要声明式部署与自愈；Prompt、Spec、路由策略也应当进入版本管理；…故而，现有的 Kubernetes 集群、CI/CD、监控与 IAM，通常不必推倒。`
  改后：`对运营模型的继承，实则是对分布式系统运营方法论的继承。云原生的运营方法论中，包括声明式期望状态、…在 AI 时代仍然有效。譬如说，Agent 的运行时和模型服务化（Model Serving）同样需要…声明式部署；提示词、规格（Spec）和路由策略也应当进入版本管理；…可以认为，AI 原生架构对基础设施的要求与云原生架构是一脉相承的。…与其费时费力地完成重建，不如有选择的继承…。`
  （要点：术语译中文 + 括号注英文、去掉中英强行空格、破折号 +「这些」回指改自然收束、平行罗列连接词改"譬如说 / 可以认为 / 与其…不如…"、起笔先回指再展开。）
