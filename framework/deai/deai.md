# 去 AI 化（De-AI）· 内部流程

> **调用规则（由写作 skill 强制执行）**：每次初稿完成后，写作 skill 必须加载本文件，
> 走完下方 **全部 7 遍**，对用户**不可见、不询问、不显示「正在去 AI 化」**，
> 完成后再输出终稿。这是不可跳过的内部步骤。

## 总原则：AI 痕迹与文体、风格正交

- **去 AI ≠ 去正式**。是否具有 AI 特征，和文本是否正式、属于哪类文体，是**两条独立的轴**。
  一篇正式论文可以既正式又"像真人写的"；一篇口语博客也可以既随意又"明显是 AI 写的"。
- 本流程只消除"**统计上的机器腔**"：低 burstiness（句长均匀）、低 perplexity（用词可预测）、
  **结构对称**（每段都是"观点→支撑→收束"）、**过渡套路**、**意义膨胀**、**模糊归因**、
  **破折号/翻译腔滥用**等。这些痕迹在任意文体里都该改。
- **完整保留**目标文体（`types/*.md` harness 决定的正式度/密度）与声音 DNA
  （`style/myself.md` 或 `style/heteronyms/*.md`）。论文去 AI 后仍是正式论文——
  只是有了真人的节奏、立场与具体细节，而不是被"口语化"伪装成人写。
- **完整保留作者的自然密度、长度与第一人称声音**。去 AI 去除的是"机器节奏"，
  **不是信息量**：领域具体例子、命名机制、数据指标、个人经验标记、结构性列举、
  Markdown 图片链接（`![[...]]`）与图表占位——即使显得"啰嗦"也要保留，不得为追求
  "精炼"而压成概述。技术 / 论文类去 AI 后仍是长而具体的。
- 不编造事实、不篡改用户原意与数据；只改表达方式。

---

## Pass 0 · 对齐目标

- 确认本次要写的 **文体**（`types/blog|article|paper.md`）与 **文字风格**
  （基础 6 + `myself.md` + `heteronyms/*.md` 之一）。
- 明确：本次只动"机器腔轴"，**不动文体轴**。正式文本保持正式，口语文本保持口语。

## Pass 1 · 检测（只标不改）

- 加载 `./ai-tells.md`，逐段通读初稿，按类别标注 AI 痕迹（**先列报告，不急着改**）：
  - 句长是否均匀（连续 3+ 句都在 15–25 字）？
  - 过渡是否套路（此外 / 然而 / 综上所述 / 值得注意的是）？
  - 是否意义膨胀、宣传腔、模糊归因（专家认为 / 行业报告显示）？
  - 结构是否对称（每段同形状）、是否连.Back to back 三段式？
  - 破折号 / 粗体 / 翻译腔（作为 / 关于 / 基于 / 进行）是否滥用？
- 输出一份**检测报告**：每条标出"类别 + 位置（第几段/哪句）"。本遍不动笔。

## Pass 2 · 删减（ROI 最高，但只删机器腔废话）

- 删掉"正确但没有信息量"的机器腔废话：复述常识、空泛总结、铺垫套话、冗余承上启下。
- **不设硬性删减比例**。草稿已干净、无可删 filler 时，本遍直接跳过，不强行凑比例。
- **「不得删」清单（即使显得啰嗦也必须保留）**：
  - 领域**具体例子**与场景（如某行业的排障、某金融场景的熔断）；
  - **命名机制 / 协议 / 方法**（如规则/语义/学习型路由、dry-run、知识飞轮、本体六要素）；
  - **数据指标**与可归因数字；
  - **个人经验标记**（"根据我个人…经验""就我个人的经验"等第一人称权威）；
  - **结构性列举**（三种路由、六要素等成组要点，不得合并压成概述）；
  - **Markdown 图片链接 `![[...]]` 与图表占位**、用户原稿中的具体条目。
- 能删则删，但凡属于上列内容的，一律不动。

## Pass 3 · 结构去痕（与文体正交，所有类型都做，力度随类型调）

- **句长刻意交替**：每段至少含 1 句 <10 字、1 句 >25 字；允许片段句。
  - blog：可多用短句、片段；article：适中；paper：可长句，但别全在 18–26 字一个节奏。
- **打破段落均匀**：拆长段、并短段、让某段只有一句、以问句或具体场景开头。
- **重写首尾段**：AI 的开头（"随着…的发展"）与结尾（"总之…很重要"）信号最重，优先改。
- **列表长度交替**：避免连.Back to back 的三段式 / rule of three；两组、四组、揉进一句都行。
- 注意：**paper 仍保持正式**，只是消除机械节奏，不改成口语。

## Pass 4 · 词汇去痕（调用 `./ai-tells.md`）

- 按目录替换 AI 高频词与过渡套话；清理翻译腔（作为 / 关于 / 基于 / 进行）。
- 克制破折号、粗体；中文正文用 `「」`。
- **替换词按文体选**：paper 用"正式但具体"的词（不用口语缩写）；blog 可用口语。
- 模糊归因若无真实出处 → 删，或标 `[需补充真实数据]`，**绝不编造**。

## Pass 5 · 声音校准

- 用所选声音 DNA（本人或异名）调：句长习惯、口头禅、转场方式、标点习惯。
- 注入"**真人决策感**"：明确立场、诚实的不确定性、具体细节与场景——
  但这些加在"机器腔轴"上，不破坏文体轴（论文照样用「笔者」「本研究」第一人称）。
- 若用户选「个人定制」：结合 `../style/myself.md` 的声音 DNA。
- 若选某「训练风格」：结合 `../style/heteronyms/*.md` 的 DNA。
- 去 AI 后的文字应更贴近该目标声音，而非千篇一律的"去 AI 腔"。

## Pass 6 · 审计 + 二次改写

- 自问：**「这版哪里还明显像 AI？」** 列 3–5 条残留痕迹。
- 针对残留再改写一遍（回到 Pass 3–5 的对应手法）。
- 这一步是多数"去 AI 失败"的补漏关键：一遍改写往往留尾巴。

## Pass 7 · 忠实度核对（不可跳过，含源素材保真）

- 事实 / 数据 / 引用未篡改；无来源论断已标「我认为 / 据称」或 `[需补充真实数据]`。
- **未编造**数字、案例、人物言论、文献。
- 逻辑链完整，无自相矛盾或跳跃；未把"用户没说过的结论"当既定事实。
- **对照源素材保真**：以 `research/idea` 产物与大纲「必须覆盖的要点」为基准，逐条核对——
  凡源中有而终稿缺的**具体信息（例子 / 数据 / 命名机制 / 个人经验 / 图占位）**，须**补回或显式标注**，
  **禁止把具体例子压缩成概述**。同时审计 Pass 2 是否误删了「不得删」清单中的内容，误删则补回。
- 终稿**可长于草稿**：当需补回被误删的具体信息 / 例子时，允许增写，这是保真而非"注水"。

---

## 护栏（restraint / anti-swap）

- 文本已经自然 → **不改它**。
- **不去"正式化"来伪装人味**：论文保持论文的正式度。
- 禁止"装人味"：错别字、脏话、强行口语、统一短句、统一不确定 —— 比 AI 味更尴尬。
- 禁止把一种痕迹换成另一种（如删了"此外"却塞满"说实话 / 讲真"当过渡）。
- 整体没有明显 AI 味，就不要硬改；过度修改会制造另一种模板感。
- **终稿可长于草稿**：当需补回被误删的具体信息 / 例子 / 图占位时，允许增写，属保真而非"注水"；
  不与"精炼度"维度冲突——保真优先于短小。

## 自评分 rubric（1–50，输出前自核，不输出给用户）

| 维度 | 标准 | 分 |
| --- | --- | --- |
| 直接性 | 直说事实 vs 绕圈宣告 | /10 |
| 节奏 | 长短句是否交替 vs 机械重复 | /10 |
| 信任度 | 尊重读者智慧 vs 过度解释 | /10 |
| 真实感 | 像真人说话 vs 机械生硬 | /10 |
| 精炼度 | 无冗余 vs 大量废话 | /10 |

标准：45–50 优秀，35–44 良好，**低于 35 回到 Pass 3 再走一遍**。

## 内部检查清单（收尾自核，不输出给用户）

- [ ] 检测报告已生成，六类痕迹已处理（Pass 1–4）
- [ ] 句长 / 段落节奏自然（不是匀速）
- [ ] 未破坏所选文体与声音 DNA（正交原则守住）
- [ ] 首尾段已重写、过渡套路已清
- [ ] Pass 6 审计二次改写已完成
- [ ] Pass 7 忠实度核对通过：未编造事实 / 数据 / 引语
- [ ] 源素材中的具体例子 / 数据 / 命名机制 / 图占位均保留或已补回（无被误删）
- [ ] 自评分 ≥ 35
- [ ] 像「真人在用该文体写作」
